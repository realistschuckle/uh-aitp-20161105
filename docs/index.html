Curly braces...

Group related statements together.

The curly braces belong to the code immediately proceeding them.

Here, the curly braces belong to the if statement. Everything inside the open and close curly braces will only run if the condition for the if statement evaluates to true 


if (someNumber < 10)
{
    // All of these statements will only run
    // if "someNumber" is less than 10.
    Console.WriteLine("Yes, you have less than 10.");
    Console.WriteLine("Don't worry. That's more than most.");
}

Here, the left-most curly braces belong to the namespace. Everything inside the namespace's curly braces belong to the namespace. In this case, the MyAwesomeClass definition will exist in the My.Awesome.Program namespace. Anything in the curly braces for MyAwesomeClass will belong to the class. 


namespace My.Awesome.Program
{
    public class MyAwesomeClass
    {
        // Some awesome class stuff in here
    }
}

Here, the curly braces belong to the method Main. All of the statements inside the curly braces will execute when some other code executes the Main method. 


public static void Main(string[] args)
{
    Pizza myPizza = new Pizza("large", "new york");
    Console.WriteLine("Your pizza will cost: " + myPizza.Cost);
}

In this code, when some other code runs Main, then a new Pizza gets created and we print its cost.

Semicolons...

End Statements

A statement is a declaration, an assignment, a calculation, or method call.

A variable is a type and a name. We declare variables so the program knows that we'll want to use them. Declarations end with a semicolon.

The highlighted code declares a variable with the type of string and with a name of myName. 


public static void Main(string[] args)
{
    string myName;

    Console.WriteLine("What is your name?");
    myName = Console.ReadLine();
    Console.WriteLine("Hello, " + myName);
}

We use the assignment operator = to assign a value to a variable. Assignments end with a semicolon.

The highlighted code shows an assignment of the myName variable with the value read from the input of the console. The statement ends with a semicolon. 


public static void Main(string[] args)
{
    string myName;

    Console.WriteLine("What is your name?");
    myName = Console.ReadLine();
    Console.WriteLine("Hello, " + myName);
}

When we call methods outside of a decision point, those end in a semicolon, too.

Note that the method call WriteLine ends in a semicolon.

Note that the method call in the if does not end in a semicolon because it's part of the decision process and not a statement all by itself. 


double myAge = Birthday.CalculateAge(myBirthday);

if (mySupervisor.IsHappyWithMyPerformance()) // no semicolon
{
    Purchase purchase = onlineStore.BuyWhiskey();
    purchase.ShipTo(mySupervisor);
}

Classes

Bundles of behavior

(Oh, and data. Don't forget data. But, behavior is more important.)

Declaring a class


public class Pizza
{
    // Some (private!) data fields
    // A constructor or two
    // Some methods that change the data in the object
    // Some properties that get and/or set data in the object
}

Fields in a class

We usually make variables for an object have private so that code from outside of the object can't mess around with the values of the data. 


public class Pizza
{
    private decimal cost;
    private int numberOfToppings;
}

Constructors

A constructor is a special method run when code creates a new instance of the class. It has no return type and the same name as the class.

Normally public, we want outside code to create new instances. 


public class Pizza
{
    public Pizza(string size)
    {
        if (size == "medium")
        {
            cost = 8.50m;
        }
        else
        {
            cost = 14.50m;
        }
    }
}

Methods

Methods of a class exist to change the value of the private data in the object and to report information about the state of the object. 


public class Pizza
{
    public void AddTopping(string topping)
    {
        numberOfToppings += 1;
        if (numberOfToppings > 3)
        {
            cost += 0.25m
        }
        else
        {
            cost += 1.00m;
        }
    }
}

Properties

Properties of a class act as a line of defense against outside code changing the data of a private variable in the object.

Using a getter in the property allows an object to hand back the value of a private variable without allowing public access.

Using a setter in the property allows for data validation before actually setting the value of private data. 


public class Pizza
{
    public decimal Cost
    {
        get { return cost; }
    }
}
